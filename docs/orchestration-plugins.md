# Оркестрация в Claude Code: шпаргалка по двум плагинам

Два плагина для оркестрации, установленных в этой среде. Они на **разных уровнях**
и не конфликтуют — можно использовать по отдельности или вместе.

| | **Senior Fable** | **claude-session-driver** |
|---|---|---|
| Что раздаёт | субагентам **внутри одной** сессии | **отдельным полным** сессиям Claude Code |
| Механизм | Agent tool (изолированный контекст) | tmux + процессы `claude` CLI |
| Главная цель | экономия токенов: дорогая модель решает, дешёвые печатают | параллельная работа над многими задачами/проектами |
| Область | одна задача, один репозиторий | несколько задач/репозиториев сразу |
| Модель | заточен под **Fable 5** (на Opus/Sonnet no-op) | не привязан к модели |
| Автор | AndyShaman | obra (Jesse Vincent) |

---

## Senior Fable

**Идея:** главная модель (Fable 5) — тимлид. Держит декомпозицию, архитектуру,
спорные решения и финальную сборку, а рутину раздаёт дешёвым субагентам.

### Кому что уходит

| Работа | Кому | Почему |
|---|---|---|
| Тесты, boilerplate, форматирование, механические правки с чёткой спекой | `fast-worker` (Sonnet) | Дёшево, суждение не нужно |
| Долгие раскопки: код, логи, многофайловые расследования | `deep-reasoner` (Opus, **только чтение**) | Изоляция контекста — «грязь» остаётся в субагенте |
| Рискованные/спорные решения | `codex:rescue` (опц.) | Независимое второе мнение |
| Декомпозиция, архитектура, ревью, синтез | сам тимлид | То, за что платишь топ-тариф |

### Как запустить

```
/model fable
/senior-fable
```

Дальше дать **объёмную многошаговую** задачу (не однострочник — на тривиальном
он намеренно не делегирует).

### Правила, которые он соблюдает

- **Least privilege:** `deep-reasoner` физически лишён Write/Edit; `fast-worker`
  получает ровно 6 инструментов.
- **Спека по шаблону:** Goal / Files (in & out of scope) / Constraints /
  Definition of done / Report format. Субагент не видит диалог — спека
  самодостаточна.
- **Acceptance gate:** результат без нужной структуры отчёта или без вывода
  верификации = проваленное делегирование, не интегрируется.
- **Один писатель на набор файлов:** параллельные делегации только с
  непересекающимися файлами.
- **YAGNI:** лишнее вырезается до маршрутизации, а не строится.

## claude-session-driver

**Идея:** один Claude (контроллер) дирижирует несколькими **настоящими**
сессиями Claude Code как работниками — через tmux, в том числе в разных проектах.

### Сценарии и как их попросить (обычными словами, без команд)

- **Delegate & wait:** «Запусти worker в проекте `~/projects/api`, задача —
  *написать тесты для auth*, дождись и покажи результат.»
- **Fan-out:** «Подними 3 worker-а: тесты / доки / линтер. Дождись всех, собери отчёт.»
- **Pipeline:** «Worker A рефакторит модуль → передай результат worker B для тестов.»
- **Supervise:** «Веди диалог с worker над фичей, показывай мне каждый его ответ.»
- **Hand off:** «Открой живую сессию worker-а в tmux — я допишу сам.»

### Механика (что под капотом)

`launch → send task → observe → collect → shutdown`, реализовано через
`tmux new-session` / `send-keys` / `capture-pane` / `kill-session`, плюс события
жизненного цикла в JSONL для отслеживания статуса.

### Требования

- Установленный **tmux** (`brew install tmux` на macOS).
- Это **полноценные автономные сессии**: тратят токены целиком и могут менять
  файлы в проектах, куда их направили. Начинать с одного worker-а на маленькой
  задаче.

## Когда что выбирать

- **Одна задача, один репозиторий, хочу сэкономить токены** → Senior Fable.
- **Много задач/проектов параллельно, хочу «менеджера»** → session-driver.
- **И то и другое:** главная Fable-сессия как тимлид + session-driver, чтобы
  разнести работу по нескольким репозиториям.
